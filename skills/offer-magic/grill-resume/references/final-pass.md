# Final Pass

Read this reference only after a validated review when the user asks whether the resume is finished, whether another micro-edit is worthwhile, or whether several target JDs are equally covered.

If the request combines review and freeze judgment, finish and validate the isolated HR review first. Reuse its cited match ranges and findings; the final pass classifies actions and decides whether to stop, but does not invent a second score.

## Classify the remaining issue

Put every proposed change in exactly one class:

| Class | Test | Action |
|---|---|---|
| Must-fix | Unsupported claim, contradiction, omitted verified hard gate, ATS parse defect, or delivered-layout defect | Fix and revalidate |
| Conditional hard gate | A true fact would directly clear a stated gate, but the packet does not establish it | Ask one focused question; edit only after confirmation |
| Wording gap | Direct evidence exists, but the JD's high-signal term or the thesis is not recoverable on first scan | Make the smallest truthful edit |
| Evidence gap | The resume shows only adjacent or no evidence for a role requirement | Keep the gap explicit; build evidence or create a separate future version |
| Optional polish | The change alters neither truth, match, scan path, nor interview defensibility | Reject the edit and freeze |

A missing keyword is a wording gap only when direct evidence already exists. Otherwise it is an evidence gap. Technical-stack names, seniority, ownership, product scope, and deployment history never become evidence through phrasing.

## Keep target JDs separate

Score each core JD independently with [../frameworks/match-rubric.md](../frameworks/match-rubric.md). Report a range and a qualitative band for each; do not average them into one flattering number. Let the core corpus determine the stable candidate narrative. Use auxiliary JDs only to detect general capability gaps and single-posting overfit.

Accept a score only when the reviewer exposes the candidate-blind effective Must Haves, per-item evidence values, zero count, raw score, cap, and final range. If the prose names two or more evidence-free core requirements but the score remains above 55%, reject the score as internally inconsistent and rerun the review before deciding whether to freeze.

One resume may be strong for one role and adjacent for another. Preserve that boundary. When the second role requires a different proof chain—such as full-stack UI delivery rather than Agent Harness mechanisms—recommend a separate evidence-backed version instead of diluting the stable resume.

## Freeze gate

Freeze the candidate version when all of these are true:

- every verified hard gate is visible;
- every retained high-signal term has direct or explicitly adjacent evidence;
- the five-second profile is clear;
- the reviewer finds credible distinctiveness, consequential trade-offs, one coherent narrative, and failure-aware proof;
- the artifact passes its frozen layout and delivery contract;
- every remaining suggestion is an evidence gap or optional polish.

State the stop decision plainly. A completed resume is not one with no conceivable edits; it is one where further wording changes have lower expected value than preserving its current evidence hierarchy.
