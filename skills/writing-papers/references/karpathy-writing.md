## 论文是一种"特定物种"

写好论文是学者的生存技能(像穴居人会生火)。要意识到:**papers 有特定的外观、流动、结构、语言与统计特征,学术同行对此有预期**。早期 draft 往往很糟——这里有大量可学的东西。

## 评审烂论文来建"二分类器"

只读好论文去蒸馏模式,等于二分类只喂正样本——不是好策略。**你更需要大量接触烂论文**,而**评审(review)**正是途径:顶会接收率约 25%,你评的多数是烂的,从中能练出一个强力的"好/坏二分类器"——亲眼看到不清晰、不定义变量、intro 空泛抽象、太快钻进细节等坑,从而在自己论文里规避。参加或组织 **journal club**,看资深研究者怎么批论文,体会自己的论文会被怎样审视。

## 先抓单一核心贡献(single)

动笔前先确定论文对领域做出的**单一核心贡献**——尤其强调**单一**。论文不是你跑过的一堆实验的随机集合,而是**售卖一个此前不显然/不存在的东西**:你要论证它重要、此前没人做过,再用受控实验支撑其价值。**全文以外科手术般的精度围绕这一个贡献组织**:不堆 fluff、不在旁边夹带别的东西。

Karpathy 亲历的反例:

- video classification 论文里硬塞了第二个(不相关的)multi-resolution 架构贡献——理由是"也许有人感兴趣"+"两个贡献优于一个"。**这是错的**:第二个贡献次要/可疑,**稀释**了论文、分散注意、没人 care。
- CVPR 2014 论文塞进 ranking 与 generation 两个独立模型——事后看更该拆成两篇。

结论:**"两个贡献优于一个"是错觉。** 一篇一个。

## gestalt:论文的特征外观

资深研究者把论文的 **gestalt(整体观感)当作强力启发式**:Karpathy 见过 Fei-Fei 把 4 篇审稿各翻 10 秒,就判出 1 好 3 坏(与他花数小时的结论一致)。要让你的论文**具备同样的外观**,因为很多人用它当认知捷径来判你的工作:

- intro 约 **1 页**;related work 约 **1 页**、**引用密度适中**(不稀疏也不拥挤)。
- 精心设计的 **pull figure**(第 1–2 页)与 **system figure**(第 3 页)——**别用 MS Paint 画**。
- technical section 里有数学符号;results table **数字多、部分加粗**。
- 一个额外的 **cute analysis 实验**。
- **正好写满页数上限**(如 8 页),一行都不少。

## 结构与段落

- 顶层默认结构:**Intro / Related Work / Model / Experiments / Conclusions**。
- 写 intro 时,先用 **latex 注释**把 top-level narrative 列成连贯骨架,再在下面填正文。
- **每段围绕一个具体的点,首句就把它点明**,其余句子支撑它——这样读者能轻松 **skim**。
- intro 的好流动:1) **X 是重要问题**(X 不显然就先定义);2) **核心挑战**是 A、B;3) **前人**用 Y 解决过,但**问题是 Z**;4) **本文做 W**;5) W 有这些**好性质**,实验证明。
- 列挑战时**只列你后面真正解决的**,不要跑题谈无关的(推测留到 conclusion)。
- **全文都要有结构,不止 intro**。例如 model 每节:1) 清楚说这节在做什么;2) 核心挑战是什么;3) baseline / 前人怎么做;4) 动机 + 你怎么做;5) 描述它。

## 反模式:laundry list(流水账)

要极力避免的结构:"这是问题。好,先做 X,再做 Y,再做 Z,再做 W,得到结果。"——**每一步都要 justify / motivate / explain**:为什么做 X 或 Y?有什么替代?别人怎么做?(常见做法可加引用。)**论文不是报告、不是你做过之事的枚举、不是把时间顺序的实验笔记翻译成 latex**;它是对"问题 + 你的方法 + 其背景"的高度处理、高度聚焦的讨论,要**教会同行**并**论证每一步**,而不只是描述你做了什么。

## 语言:good words vs bad words

会逐渐形成一套好词/坏词表(以 ML / CV 论文为例):

- 不要 "**study**" / "**investigate**"(被动、无聊、坏词)→ 用 "**develop**",更好是 "**propose**"。
- 不要 "**system**",更别说 "**pipeline**" → 用 "**model**"。
- 不 learn "**features**" → learn "**representations**"。
- **绝不** "**combine**" / "**modify**" / "**expand**"——这些是增量、廉价的词,几乎注定让论文被拒 :)。

> 这与"去 AI 味"互补:去 AI 味删的是空洞自夸/机械连接,这里换的是**增量感、被动感**的动词。

## 与 Widom 两套"五点"的衔接

Widom 的引言五段(**问题 / 为何重要 / 为何难 / 为何此前未解决 / 方法与局限**)与 Karpathy 的 intro flow(**重要问题 / 核心挑战 / 前人 Y 及其问题 Z / 本文 W / 好性质 + 实验**)是**同一件事的两种措辞,不冲突**:都要求引言把"问题 → 为何难 / 前人不足 → 你的方法 → 证据"讲清。任选其一打骨架,再用 Karpathy 的「每段首句点题、不要 laundry list、good/bad words」打磨语言与段落。
