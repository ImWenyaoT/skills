# Layout principles (tool-agnostic)

What good resumes share, independent of the tool you render with. **Master test: shrink the
finished PDF until you can't read words — every section should still read as its own distinct
block.** If it does, the layout works.

## Type and weight

- **Sans-serif, light body:** a light body weight (≈300) means **bold (≈700, pure black) snaps the
  eye instantly** — a contrast serif/regular body can't match.
- Emphasis (name, project names, **key numbers**) in bold + pure black.

## Size

- Keep body **10–14pt**: body ~10.5, section headings ~13.5, project names ~11 (name can be larger, ~24).
- ≤ 3 size tiers. Don't get fancy.

## Spacing and rhythm

- Line height **~1.5** (breathable, readable).
- **Loose between blocks, tight within a block.** Make the gaps decrease into a hierarchy:
  before a section heading (~7mm) > between entries (~3.5mm) > between bullets (~1.2mm). This
  decreasing gap makes "small things small, big things big" so each block reads as one shape.

## Boundaries and color

- Each section gets a **clear boundary:** bold heading + a full-width rule; a heavier rule under
  the header.
- **Minimal color:** black/dark-grey; layer with weight + spacing + rules, not hue.

## The shrink test (do this last)

Shrink the PDF small and glance: are the sections (contact / education / experience / projects /
skills / summary) each a clear, separate block? Yes → done. Blurs into one mass → increase
inter-block spacing and strengthen heading contrast until they separate.
