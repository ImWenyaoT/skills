# Self-check checklist

Run the whole resume through this when done. ✅ = must satisfy.

## Format (whatever tool renders it — still verify)

- ✅ Page count under control: aim for **1 page** for most candidates (2 only with deep
  experience). Cut by relevance, not by shrinking type to cram in irrelevant content.
- ✅ Body in a light/regular weight with **bold for emphasis** so key items pop; ≤ 3 type sizes;
  sane margins.
- ✅ Mostly black/grey, minimal color; rely on weight + spacing + rules for hierarchy, not color.

## Contact / education

- ✅ Header essentials present: name / phone / email / target role (+ years of experience or
  student status). Add a portfolio/GitHub link if relevant.
- ✅ Education: school · dates · major. Add GPA/rank/coursework **only if they help** (strong GPA,
  relevant courses); otherwise omit.

## Experience / projects (the core)

- ✅ Work/internship first, academic second — unless a newcomer with no formal work, then
  **projects carry the resume** and papers go under research.
- ✅ Each entry has a **descriptive role** ("independently designed and built", "author/maintainer"),
  not just a project name.
- ✅ Every bullet **starts with a strong verb**; no verb repeats within one entry (see verb-list.md).
- ✅ At least one bullet per entry carries a **quantified result** (number / % / scale / N items).
- ✅ Industry keywords embedded naturally (helps ATS).
- ✅ **No restatement:** bullets don't re-describe the same thing (check each entry's first vs last bullet).
- ✅ **No filler verbs:** grep for "responsible for / involved in / participated in / worked on"
  → should be 0.

## Honesty (hard gate)

- ✅ No fabricated numbers; old metrics labeled "(snapshot)".
- ✅ Paper/project status truthful (published / under review / in progress); second author not
  written as lead/first-author.
- ✅ No piling model names as if they were skills.

## Scope discipline (don't add back what was deliberately cut)

- Decide up front what's **out of scope** for this resume (e.g. a language version you won't write,
  extra file formats, a hobbies section) and don't reintroduce it bullet by bullet.
